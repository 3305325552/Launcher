# Launcher 使用手册

Launcher 是一个本地优先的 Windows 快速启动器，用来整理常用程序、文件夹、网页、脚本、命令、笔记和插件搜索结果。它的目标是快速唤醒、快速搜索、低内存占用，并尽量减少对前台工作的打扰。

## 快速开始

运行 `launcher.exe` 后会自动创建配置。配置默认保存在：

```text
%LOCALAPPDATA%\Launcher\config.json
```

可以在 `设置 > 常规 > 数据` 中打开配置目录、切换配置目录、导入或导出配置。

## 主窗口

左侧是分类栏，右侧是当前分类的项目区域。

- 单击分类切换分类。
- 双击项目运行。
- 右键项目打开操作菜单。
- 拖入文件可以添加到当前分类。
- 中键点击分类可以运行该分类下的全部项目。
- 关闭按钮默认隐藏到托盘。

项目支持程序、URL、脚本、标题、占位符、虚拟文件夹和笔记。新建虚拟文件夹会直接打开项目编辑窗口，和新建空项目一致，便于先修改名称和属性再保存。

## 搜索

打开搜索后直接输入关键词。空搜索和加载中状态会显示在搜索区域中央；加载时会显示轻量动画。

搜索支持：

- 名称、关键词、备注匹配。
- 正则表达式。
- 拼音首字母和拼音匹配。
- 英文模糊匹配。
- 全局文件搜索。
- 插件搜索结果。

只搜索启动器内部项目：

```text
qs 关键词
```

只搜索指定目录下的全局文件：

```text
dir C:\Users\Alice\AppData\Roaming ddnet
dir "D:\Path With Space" keyword
```

`dir` 指令只使用全局文件索引，不会实时扫描磁盘。目录后面的关键词仍然使用全局搜索的排序和模糊匹配；目录路径包含空格时需要用引号包住。

搜索结果快捷操作：

- `Up / Down`：选择上一条或下一条结果。
- `Left / Right`：上一页或下一页。
- `Alt+1` 到 `Alt+0`：运行当前页前 10 条结果。
- 右键结果可打开上下文菜单。

## 交互式项目

项目可以启用交互式运行。双击后会弹出参数窗口，可输入文本、调节数字或从预设选项中选择。输入框支持历史参数：

- 聚焦输入框时显示历史。
- 历史按使用次数和最近使用时间排序。
- `Up / Down` 选择历史项。
- `Tab` 轮换历史项。
- `Enter` 补全或运行。
- 每条历史右侧可以删除。

适合把常用 CLI 命令做成带参数的快速入口。

## 任务计划

右上角主菜单中的 `任务计划` 可以创建自动运行任务。任务会在 Launcher 运行时检查和触发，配置保存在本地配置文件中。

支持触发方式：

- `一次`：从现在开始延迟指定分钟后运行一次，运行后自动禁用。
- `每天`：每天指定时间运行。
- `每周`：在选中的星期和指定时间运行。
- `间隔`：每隔指定分钟运行。
- `启动`：Launcher 启动后运行一次。
- `唤醒或解锁`：系统会话解锁或登录后运行。
- `进程启动`：检测到指定进程名从未运行变为运行时触发。

支持动作：

- 运行已有项目。
- 运行虚拟文件夹内的可运行项目。
- 最小化运行、失败重试、错过任务是否补跑。

任务页会显示下次运行时间、上次运行时间、最近结果和历史记录，也可以手动 `立即运行`。

## 插件

插件支持两种形态：进程外 exe 插件和进程内 native DLL 插件。exe 插件通过 stdio JSON 与 Launcher 通信，隔离性更好；native 插件通过 C ABI 加载 DLL，速度更快但运行在 Launcher 进程内，崩溃会影响主程序。

插件目录：

```text
<launcher.exe 所在目录>\plugins\<plugin-id>\
<配置目录>\plugins\<plugin-id>\
```

每个插件目录需要包含 `plugin.json`。支持能力：

- `search`：返回搜索结果。
- `actions`：执行搜索结果的右键动作。
- `events`：接收 `app.started`、`content.changed` 等事件。
- `tasks`：读取任务计划快照，并通过动作返回任务创建、更新、删除、启用、禁用或立即运行请求。
- `settings`：在设置页显示插件自己的配置项。

插件页位于 `设置 > 插件`。可以启用、禁用、重新加载插件，或打开插件目录。测试构建会自动打包几个示例插件：

- `Sample Echo Plugin`：搜索 `tp`。
- `Sample Calculator Plugin`：搜索 `calc 12.5*4`。
- `Sample Color Plugin`：搜索 `color #336699`。
- `Sample CLI Preview Plugin`：搜索 `cli status --short`。
- `Sample Native Task Plugin`：搜索 `native task`，用于测试 DLL 插件和任务计划 API。

## 设置

设置窗口左侧是分类，顶部提供设置搜索。输入关键词后会显示匹配的设置项，点击结果会跳转到对应页面。

设置页包括：

- `常规`：语言、开机启动、全局唤醒热键、配置目录、导入导出。
- `动作`：双击运行、运行后隐藏、虚拟文件夹运行行为、空白区域拖动窗口、窗口自动隐藏。
- `界面`：主题、动画、窗口锁定、默认项目布局、工具提示。
- `动作 > 窗口 > 拖动空白区域移动窗口`：允许按住主窗口空白区域移动窗口；标题栏拖动不受这个设置影响。
- `搜索`：搜索延迟、全局搜索、高级搜索、默认图标、匹配模式。
- `插件`：插件列表、启用状态、插件配置。
- `帮助`：只读 Markdown 用户手册预览。
- `关于`：程序和运行环境信息。

右上角主菜单中的 `帮助` 也会打开同一份本地用户手册预览。构建时会自动把 `docs/USER_GUIDE.md` 同步到 `assets/docs/USER_GUIDE.md`，不需要手动复制。

## 动态背景

主题编辑器支持静态图片、GIF 和常见视频文件作为主窗口背景。在主题编辑器的 `背景文件` 行点击 `选择文件`，可以选择图片、GIF 或视频。选择 GIF/视频后会自动启用 `动态背景`，再点击 `生成缓存`，Launcher 会调用系统 `PATH` 中的 `ffmpeg` 把源文件预处理为 JPEG 帧。

缓存保存到当前配置目录下：

```text
<配置目录>\background-cache\
```

推荐参数：

- `动画帧率`：12 到 15 FPS，视觉上足够平滑且 CPU 占用低。
- `动画最大宽度`：960 或 1280，窗口较大时可提高。
- `JPEG 质量`：数值越小质量越高、缓存越大；默认值偏向低内存和低磁盘占用。

播放时只保留当前帧和下一帧纹理，窗口隐藏或不绘制时不会继续加载帧。更换源文件、帧率、宽度或质量后需要重新生成缓存。

## 全局文件搜索

在 `设置 > 搜索 > 启用全局搜索` 中开启。开启后 Launcher 会在后台持续维护本地文件索引。

索引策略：

- 热路径（用户主目录 `%USERPROFILE%`）约每 10 分钟全量刷新一次，新建文件夹/文件会较快出现在搜索结果中。
- 冷路径（各本地磁盘根）按 root 滚动刷新，默认约每 12 小时覆盖一轮，避免整盘频繁重扫。
- 索引按 root 分区缓存到配置目录下的 `global-search-roots/`；旧版单文件 `global-search-cache.tsv` 会在升级后自动迁移。

相关设置：

- `Use default icons`：使用轻量默认图标，降低内存占用。
- `Global files per page`：限制每页全局文件结果数量。
- `Hide system search results`：隐藏系统目录、缓存目录等噪声结果。
- `Search delay`：输入后延迟多少毫秒再刷新搜索。

首次开启全局搜索时索引会逐步建立；之后后台会持续小量更新，无需手动重建。

限定目录搜索：

```text
dir 目录 关键词
```

例如 `dir C:\Users\Alice\AppData\Roaming ddnet` 会只在该目录及子目录中搜索 `ddnet`。如果目录本身包含空格，使用 `dir "D:\Path With Space" keyword`。省略关键词时会列出该目录范围内已索引的文件和文件夹。

## 快捷键

默认全局唤醒热键：

```text
Ctrl+Alt+Space
```

可选搜索热键：

```text
Ctrl+Alt+F
```

主窗口常用快捷键：

- `Space`：进入搜索。
- `Esc`：退出搜索或清空选择。
- `Enter`：运行当前选中项目。
- `Delete`：删除当前选中项目；没有选中项目时可删除当前分类。
- `Ctrl+A`：选择当前分类全部项目。
- `Ctrl+C` / `Ctrl+X` / `Ctrl+V`：复制、剪切、粘贴项目。
- `Alt+Up / Alt+Down`：切换分类。

## 建议

- 常用程序放进启动器分类，搜索排序会优先考虑内部项目和使用次数。
- 全局搜索适合找文件，不建议把每页全局文件结果设得过大。
- 需要固定结构时，用分类和虚拟文件夹组织。
- 需要快速定位时，补充项目关键词和备注。
